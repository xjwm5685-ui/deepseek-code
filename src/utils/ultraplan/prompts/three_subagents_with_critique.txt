## Process

### Step 1: Parallel Exploration (3 agents)
Spawn 3 agents simultaneously with distinct missions:

**Agent A — Architecture**
- Read relevant code paths end-to-end
- Understand current architecture, patterns, conventions
- Identify reusable utilities and extension points
- Deliver: architecture summary + pattern guide

**Agent B — Inventory**
- Find every file that will need changes
- Classify each: CREATE | MODIFY | DELETE | REVIEW
- Deliver: complete file list with classifications

**Agent C — Risk**
- Identify: breaking changes, regressions, security concerns, performance impacts
- Find edge cases the design might miss
- Map dependency ordering
- Deliver: risk register + dependency graph

### Step 2: Synthesize
Combine all 3 reports into a unified plan:

1. **Summary** — what changes and why
2. **File-by-file change list** — exact changes, order, reasoning
3. **Implementation sequence** — step by step with dependencies
4. **Testing strategy** — unit + integration + verification per step
5. **Risk register** — what could go wrong, how to prevent

### Step 3: Critique (1 agent)
Spawn a critique agent to review the draft for:
- Missing steps or files
- Incorrect assumptions
- Error/edge case gaps
- Ordering mistakes
- Testing gaps

### Step 4: Final Plan
Incorporate critique. Be specific. File paths, function names, type sketches.
